MANEJO DE DATOS (ESTRUCTURAS Y FUNCIONES)

Datos

¿Qué es un dato? Es una porción de información muy concreta y especifica que nos permite, a través de diferentes operaciones y estructuras, tomar decisiones. Hacen parte de un algoritmo en una estructura más grande. Son la base sobre la cual partimos y sobre la cual llegamos, además, componen todas las estructuras para que puedan funcionar y sean operables.

Datos Texto

Están compuestos por letras, caracteres y símbolos especiales. Tanto una palabra como una frase son cadenas de texto. EJ: Ana. Los espacios en la frase también forman parte de la cadena texto. EJ: Ana está estudiando.

Podemos tener 2 cadenas de texto separadas y unirlas mediante la concatenación. Para concatenar necesitamos que ambas partes sean del mismo tipo de dato, en este caso de tipo texto.

  • Parte 1: Ana
  • Parte 2: está
  • Concatenación. Anaestá

Para que nuestra cadena de texto tenga sentido lógico es necesario concatenar un espacio.

  • Parte 1: Ana
  • Parte 2: _ (el guión bajo es solo para representar un espacio)
  • Parte 3: está
  • Concatenación: Ana está

Al momento de definir tu tipo de dato como texto, todo lo que incluya tu cadena será reconocido como texto, incluso si su representación gráfica es un número, por ejemplo, un nombre de usuario: “user2004” <=es una cadena de texto.

Datos Numericos

Los datos tipo numérico pueden ser los siguientes:

  • Entero o int: Incluye los números sin ningún tipo de adición decimal. Ejemplos: 7, 10, 6872
  • Float o double: Incluye los números con parte decimal. Ejemplos: 3.141592, 19.324, 124.2454

Los operadores aritméticos nos sirven para hacer operaciones matemáticas con los datos numéricos. Ejemplo:

  • 9 + 12 = 21
  • 9 * 12 = 108

Para hacer una concatenación de números deberían ser convertidos a texto. Ejemplo: Ana07 concat 76.

  • Parte 1: ana07
  • Parte 2: 76
  • Concatenación: ana0776 (numero se convierte en texto para hacer la concatenación).

Nótese que lo anterior es distinto que realizar una suma 7 + 76 = 83 <= número

  • Ejemplo 75 concat 10 => 7510
  • Ejemplo 75 + 10 = 85

Datos Booleanos

Datos tipo booleano: Este tipo de datos representa una cantidad lógica que es una respuesta a una pregunta de verdadero o falso.

Su representación:

  • True= 1
  • False= 0

Si la condición se cumple sería true y si no se cumple sería false. En este caso del diagrama de flujo nos podría llevar a otro proceso o incluso al final.

Arrays

Los arrays son como contenedores con distintos espacios, espacios disponibles para un dato distinto, no estarían juntos como cuando son concatenados o bajo una operación matemática o datos para tomar decisión como los booleanos de verdadero y falso... Son simplemente contenedores que tienen cualquier tipo de dato: texto, numero, booleano, numero decimal, valor vacío, etc. Es una unidad que contiene otras unidades.

La primera posición de un array es 0, para entender mejor tenemos un ejemplo:

  • Array: [ "casa", 70, true, 70.1 ]
  • Posición: casa es 0, 70 es 1, true es 2 y 70.1 es 3.

Los array tienen longitud, y eso es cuanto espacios tienen en él, en este caso su longitud es de 3.

Estructuras de control

Las estructuras de control permiten tomar decisiones y realizar un proceso repetidas veces. Se trata de estructuras muy importantes, ya que son las encargadas de controlar el flujo de un programa.

Condicionales: Permiten tomar decisiones a partir de una comparación u operación.

  • if / else (se basa en resultados con true false).
  • switch (permite comparar).
  • try / catch (plan b cuando una instrucción no se pudo ejecutar correctamente).

Ciclos: Realizan diferentes iteraciones de acuerdo a condiciones o situaciones que pide el algoritmo.

  • for (hace un recorrido de acuerdo a una condicion).
  • while, do-while (realizan ciertas acciones mientras una situación este sucediendo o se cumpla y a partir de ello se ejecutaran instrucciones que estan dentro de ese ciclo).

Condicionales

Condicional if: Se usa para tomar decisiones, este evalúa una operación lógica, es decir una expresión que de como resultado True o False. Si la condición devuelve un true se ejecutarán ciertas instrucciones. Ejemplo:

  • if (3 >2)
  • "Mostrar un mensaje"

Mostrar un mensaje es la consecuencia. por otro lado nótese que acá la condición es true.

Condicional else: Si no se cumple ninguna condición se ejecutará otro bloque de instrucciones. Ejemplo:

  • if (1>2) "Mostrar un mensaje"
  • else "Mostrar el #5"

Nótese que acá la condición false por eso agregamos else.

Condicional switch: La estructura selectiva switch selecciona una de entre múltiples alternativas, es decir cuando tienes muchos resultados y dependen de condiciones distintas. Esta estructura es especialmente útil cuando la selección se basa en el valor de una variable simple o de una expresión simple denominada expresión de control o selector. Ejemplo:

  • Switch (3>2)
  • Case true: consecuencia1
  • Case false: consecuencia2

(En este caso seria mejor if else, pero si hubiera mas consecuencias es mejor switch.)

Es importante que el uso de switch tenga alfinal un valor por default, entre en caso de que ninguno de las casos suceda y tendrá una consecuencia.

Default: Sirve para que el flujo de nuestro algoritmo no se rompa en caso de que no se cumpla ningún case, mostrando un mensaje por defecto. Ejemplo completo con switch y default: "Se solicita número del 1 al 4"

  • switch (número)
  • case 1: "El número ingresado es 1"
  • case 2: "El número ingresado es 2"
  • case 3: "El número ingresado es 3"
  • case 4: "El número ingresado es 4"
  • default: "El número ingresado no está en el rango 1-4"

Ciclos

Un bucle o ciclo, es una secuencia de instrucciones de código que se ejecuta repetidas veces, hasta que la condición asignada a dicho bucle deja de cumplirse.

Ciclo while: Se utiliza para ejecutar ciertas instrucciones repetidamente mientras que se cumple una condición determinada.

Ciclo for: Se recorre desde un inicio hasta un limite mientras sea verdadera la condición. Tiene un punto de partida (punto a), cantidad de iteraciones (cantidad de veces que va a volver al mismo sitio inicial, o sea vueltas) y por ultimo el incremento (marca cuantas iteaciones faltan para completar el llegar al punto b)

Ciclo do while: Es una variante especial del bucle while . Al contrario que el bucle while , que comprueba la condición antes de entrar en el bucle, el bucle do - while la evalúa al final del bucle. Esto implica que el bucle se ejecutará al menos una vez.

¿Cuándo utilizar un ciclo for, while o do while?

  • For: cuando sabes (o puedes saber) las veces que repetirás el ciclo. Ejemplos: “5 veces”, “la cantidad de elementos que tiene un arreglo”.
  • While: Cuando no sabes las veces que se repetirá un ciclo. Ejemplos: “reintentar conectarme a una base de datos si falló al hacerlo”
  • Do while: Cuando no sabes las veces que se repetirá un ciclo y necesitas que se realice por lo menos una vez. Ejemplos: “Conectarme a la base de datos, si falló, repetir hasta que me pueda conectar”

Try catch

Instrucción que verifica y cuida la ejecución de los procesos, en caso de error ejecutará un plan de contingencia. Cuida la lógica en caso de que una instrucción haya fallado. Ejemplo:

  • Try
  • Mostrar imagen
  • (en caso de que no se pudo mostrar) -> error
  • Plan de contigencia: mostrar 🙁

Otro ejemplo:

  • Try: escribir en un documento una lista de nombres: Ana, juan pablo julia
  • En caso de que escribir estos nombres fallo por algo, se dispara: Catch .guardame en un array estos nombres

Funciones

Función: Una función es un bloque de instrucciones que hace una actividad especifica de forma independiente al resto del algoritmo. Con una función puedes segmentar partes de tu algoritmo, las cuales puedes reutilizar.

Una función tiene componentes importantes:

  • La palabra reservada function para especificar que es una función, esto puede variar dependiendo el lenguaje de programación.
  • El nombre de la función, que representa la acción que resuelve.
  • Los parámetros están envueltas en paréntesis (), son variables propias de la función y deberán utilizarse en el contenido. Valor que necesita la instrucción interna para operar. Son opcionales, es decir, puede existir 0 hasta infinitos parámetros.
  • El valor retornado es un único valor que devuelve la función cuando es invocada. Se lo especifica por la palabra reservada return.

La invocación de la función está constituida por dos componentes:

  • El nombre de la función especificada en la declaración.
  • Los argumentos, son los valores para cada uno de parámetros de la función envueltos entre paréntesis. nombre(argumentos)

Nota para distinguir parametro de argumento: En mi ejemplo (a ,b) son identificadores, o sea parametros y deben tener un valor, entonces si mando a llamar la funcion deberia poner un valor y ahí es cuando pasa a ser un argumento y no un parametro. Aquí un ejemplo para más claridad...

  • function suma (a, b) => aqui son parametros y se nombra la funcion.

    suma = a + b

    return suma

  • suma ( 5, 10) => aqui estarian de argumento y aqui se manda a llamar.

Alcance de funciones

Privadas🔒: Funcionan de forma local, no pueden ser llamadas en otros algoritmos, por eso es importante determinar el alcance de la función, ya que puedes estar tratando con información más compleja que es especifica para ese algoritmo, y no puede ser compartida a otros algoritmos que la puedan necesitar.

Públicas🔓: El alcance es global. Puedes mandar a llamar funciones de un algoritmo a otro.
